
\nchapter{Требования к~выполнению лабораторных работ}

\nsection{Операционная система и~компилятор}

Все задания, если это не сказано особо, выполняются под управлением операционной системы \stdOs{}.
Задания всех лабораторных работ, кроме \ref{lab:digits} и~\ref{lab:disasm}, если не указан язык, выполняются на ассемблере, в~виде вставок в~программу на языке высокого уровня  либо отдельных модулей. %либо в~виде отдельного проекта.

Программы на чистом C++ (без ассемблерных вставок) должны собираться на любой платформе любым компилятором
(недопустимо использование платформозависимых элементов).

Для разработки программ на языке C++ могут быть использованы кроссплатформенные среды Code::Blocks, Qt Creator, TheIDE (Ultimate++), Codelite.
% 
Для сборки и~отладки программ в~среде Microsoft Windows с~использованием компилятора Microsoft используется среда Microsoft Visual Studio.

копировать в~отчёт не нужно (текст программ предоставляется отдельно).

Отчёт в~формате plain text может быть совмещён с~программным кодом (помещён в~комментарии соответствующих модулей).

В~начале каждого файла с~исходным кодом должен находиться \emph{комментарий}, содержащий краткое описание модуля.
Описание модуля с~головной функцией должно включать \emph{тему лабораторной работы,} а~также \emph{тексты задания и~варианта.}


\nsection{Парная работа}

Лабораторная работа выполняется совместно двумя сидящими рядом студентами.
Состав пар на разных занятиях может различаться.
При нечётном количестве присутствующих допускается не более одной команды, состоящей из трёх или одного человека.
Команда, независимо от количества участников, выполняет один вариант задания, соответствующий
% (если в~задании не сказано иначе). 
% Номер варианта равен 
номеру компьютера одного из участников (по выбору).

\nsection{Оценивание}

Максимально возможная оценка лабораторной работы "--- 11 баллов.
Работа, выполненная не полностью (в~том случае, если не выполнено или выполнено с~ошибками менее 25\% обязательных заданий), %или с~ошибками,
может быть зачтена с оценкой ниже максимальной.
Работа, выполненная не самостоятельно, \emph{не может быть зачтена.}

% Минимально возможная оценка лабораторной работы в~системе ОРИОКС "--- 5 баллов. 
% % Лабораторная работа, планируемая оценка которой (с~учётом штрафа за опоздание и~дополнительных баллов за бонусные задания) менее минимально возможного балла ОРИОКС, оценивается как 0~баллов.
% Если рассчитанная с~учётом штрафа за опоздание и~дополнительных баллов за бонусные задания оценка оказывается менее 5 баллов, то система ОРИОКС назначает за работу 0~баллов. 

Из оценки
лабораторной работы, сданной с~опозданием более чем на одно занятие без уважительной причины,
\emph{вычитается величина опоздания,} то есть лабораторная работа, сданная с~опозданием на два занятия, может быть оценена (без учёта бонусных заданий) максимум на 9 баллов, с~опозданием на три занятия "--- максимум на 8 баллов, и~т.\,д.

Лабораторная работа% (кроме последней)
, сданная на зачётной неделе, оценивается как 0~баллов
(не зачтённые лабораторные работы помечаются в~ОРИОКС литерой~«н»).

\nsection{Необязательные (бонусные) задания}
Задания, отмеченные как \termin{«Бонус»}, \emph{необязательны;} за их выполнение начисляются дополнительные баллы. 
Баллы за необязательные задания добавляются к~оценке за лабораторную работу (если это возможно без переполнения) или к~колонке «бонус%».%
/зачёт».

Бонусные задания могут быть сданы либо одновременно с~соответствующей лабораторной работой, либо (при условии, что это не помешает преподавателю принимать у~других студентов обязательные задания) после неё.
На~зачётной неделе бонусные задания не принимаются.
% (кроме случаев продления семестра).

В~тексте задания указано \emph{максимальное} количество дополнительных баллов.
Количество баллов, начисленных за бонусные задания, зависит только от качества выполнения задания, но не от времени сдачи.
